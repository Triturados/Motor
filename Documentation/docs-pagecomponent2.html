LOVE Components

Slider

Clase que contiene el componente Slider de los objetos.

#include <Slider.h>

Ejemplo de uso del componente en Lua:

local sliderBehindLive = player:addComponent("Slider"):sendMsg([[
		materialBar: Heal;
		materialBarBg: Heal_bg;
		materialButton: CircleButton;
		width: 300;
		height: 50;
		posX: 100;
		posY: 100;
	]])

Nota

En el codigo mostrado se entiende que existe el objeto player.

Valores que se le pueden enviar por Lua:

materialBar Material del slider.
materialBarBg Material que esta por detras del slider.
materialButton Material del boton que indica donde arrastrar el slider
width Ancho del slider.
height Altura del slider
posX Posicion x del slider, por defecto 0.
posY Posicion y del slider, por defecto 0.
posZ Posicion z del slider, por defecto 1.

Funciones publicas disponibles desde este componente:

void receiveMessage(Utilities::StringFormatter& info)) Recibe la información enviada por Lua.
void setVisibility(bool mode) Establece al visibilidad del slider segun el boleano mode, por defecto true.
inline int getProgress() Devuelve el progreso del slider, su valor maximo es 100.
inline int getBarWidth() Devuelve el ancho del slider.
inline int getMaxBarWidth() Devuelve el tamaño maximo del ancho de la barra.
void setProgress(int progress) Establece el valor del slider segun el valor de progress.
void setDetectInput(bool mode) Estable si se detecta input o no segun el valor del boleano mode.
void setPos(Utilities::Vector3 pos_) Establece el valor posicion del slider.
void setDimensions(Utilities::Vector2 dimensions_) Establece las dimensiones del slider, siendo la componente x el ancho y la componente y la altura.

RigidBody

Clase que contiene el componente Slider de los objetos.

#include <Slider.h>

Ejemplo de uso del componente en Lua:

player:addComponent('Rigidbody'):sendMsg([[
		trigger: true;
		mass: 0.0;
		shape: cube; 
		restitution: 0.9;
		colliderScale: 10,1,10;
	]])

Nota

En el codigo mostrado se entiende que existe el objeto player.

Valores que se le pueden enviar por Lua:

trigger Boleano para indicar si el objeto va a ser trigger o no.
shape String para decir la forma del RigidBody que puede ser cube, sphere, plane o capsule.
restitution Restitucion del objeto que tiene el componente.
colliderScale Vector3 para establecer el tamaño del collider.
state String que dice como puede ser el RigidBody kinematic, dynamic o static.

Funciones publicas disponibles desde este componente:

void addForce(Utilities::Vector3 vel, Utilities::Vector3 relativePos, ForceMode mode) AYUDA

void setMass(float mass_) Pone la masa del Rigidbody al valor de mass_.
void setShape(std::string nameF_) Establece la forma del rigidbody en funcion del string nameF_.
void setLinearVelocity(Utilities::Vector3 vel) Establece la velocidad lineal del objeto segun el vector3 vel.
void setAngularVelocity(Utilities::Vector3 vel) Establece la velocidad angular del objeto segun el vector3 vel.
void setTrigger(bool trigger_) Decide si el rigidbodyes trigger o no en funcion del boleano trigger_.
void setRBGravity(Utilities::Vector3 newRBGrav) Estable la gravedad del objeto segun el vector3 newRBGrav.
void setRestitution(float resti) Establece el valor la variable restitution.
void setFriccion(float friction) Establece la fricción del objeto segun el valor de friction.
void setLinearFactor(Utilities::Vector3 vect) Establece el factor lineal segun el vector3 vect.
void setAngularFactor(Utilities::Vector3 vect) Establece el factor angular segun el vector3 vect.

Utilities::Vector3* getVelocity() Devuelve un vector3 con la velocidad del gameobject.

ParticleSystem

Clase que contiene el componente ParticleSystem de los objetos.

#include <ParticleSystem.h>

Ejemplo de uso del componente en Lua:

local pSys = particleSys:addComponent("ParticleSystem")
	pSys:sendMsg([[
		particleName: bomb; 
		emitting: true
	]])

Nota

En el codigo mostrado se entiende que existe el objeto particleSys.

Valores que se le pueden enviar por Lua:

particleName Nombre de la particula
emitting Boleane que dice si se emiten o no las particulas.

Funciones publicas disponibles desde este componente:

void receiveMessage(Utilities::StringFormatter& info)) Recibe la información enviada por Lua.
void setActive(bool activated) Activa o desactiva el sistema de particulas por defecto el boleano activated es true.
bool isEmitting() Devuelve true si el sistema de particulas esta emitiendo y false en caso contrario.
void setMaterial(std::string materialName) Cambia el material del sistema de particulas al enviado por el string materialName.
